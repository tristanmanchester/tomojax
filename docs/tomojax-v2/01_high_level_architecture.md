# 01 — High-Level Architecture

## One-sentence design

TomoJAX should be a **self-calibrating differentiable tomography and laminography engine**: it reconstructs a latent volume, differentiably optimises geometry and per-view pose against projection residuals, canonicalises gauges, verifies the result, and produces a crisp final reconstruction.

It should not be a collection of traditional alignment algorithms.

## Product identity

The public product should feel like this:

```bash
tomojax preprocess raw.nxs corrected.nxs
tomojax align corrected.nxs --mode auto --quality fast --out aligned.nxs
tomojax recon aligned.nxs --out recon.nxs
```

Optional high-level profiles:

```text
--mode cor           # solve detector-centre/setup bootstrap
--mode pose          # solve per-view pose using current setup
--mode auto          # setup + pose + verification
--mode max           # slower, verification-triggered full-resolution polish

--quality fast       # quick product default
--quality reference  # slower, conservative evidence run
```

Avoid exposing method choices such as `--cor-grid`, `--roll-search`, `--phase-correlation`, or `--spsa`.

## Core objective

Use one model:

```text
minimise over:
    x   = reconstructed attenuation volume
    g   = global setup geometry
    p_i = per-projection pose / stage residuals
    m_i = optional object-frame motion/deformation
    n_i = nuisance acquisition state

sum_i robust_projection_residual(
    forward_project(x; g, p_i, m_i, n_i) - y_i
)
+ reconstruction_prior(x)
+ setup_prior(g)
+ pose_prior(p)
+ motion_prior(m)
+ nuisance_prior(n)
```

In equation form:

\[
\min_{x,g,p,m,n}
\sum_i \rho \left(
W_i \left[
F_i(x; g,p_i,m_i,n_i) - y_i
\right]
\right)
+ \lambda_x R(x)
+ R_g(g)
+ R_p(p)
+ R_m(m)
+ R_n(n)
\]

where `F_i` is the differentiable acquisition model for projection `i`.

## Default global setup parameters

The setup block should represent physical acquisition geometry, not alignment stages:

```text
det_u_px                 # detector horizontal centre / COR-equivalent term
det_v_px                 # optional; observability-gated
detector_roll_deg
axis_rot_x_deg
axis_rot_y_deg
theta_offset_deg
theta_scale              # optional; observability-gated
theta_nonlinear_model    # optional spline/drift model
detector_pixel_scale     # optional / future
detector_pixel_anisotropy# optional / future
beam_direction convention
detector coordinate convention
```

## Default per-view pose parameters

Per projection:

```text
alpha_i
beta_i
phi_residual_i
dx_i
dz_i
```

These are residual pose terms, not necessarily instrument truth. The realised forward geometry is what matters for reconstruction.

## Object-frame motion

Object-frame motion is separate from detector/stage pose. This matters for thermal drift, glue expansion, shrinkage, or sample creep.

Initial object-motion model:

```text
tx_obj_i
ty_obj_i
tz_obj_i
small rotations, optional
```

Future object-motion models:

```text
affine object drift
low-rank object-frame deformation
B-spline deformation field
time-parameterised deformation
```

Object-frame motion should be an escalation path, not a default for simple scans.

## Nuisance acquisition state

A general toolbox must model nuisance terms so that geometry is not forced to explain intensity problems.

Start with:

```text
per-view gain
per-view offset
background offset
low-frequency background field, optional
bad-pixel mask
saturation mask
valid detector mask
```

Future nuisance terms:

```text
stripe/ring bias fields
flat-field drift
partial-FOV validity model
scatter surrogate
beam-hardening surrogate
```

Nuisance terms should have priors and be solved internally, not exposed as separate alignment stages.

## Geometry graph

Do not hardcode a single formula for every experiment. Build a typed geometry graph:

```text
lab frame
  ├── beam frame
  ├── detector frame
  ├── stage frame
  └── object frame
```

A projection is generated by composing transforms:

```text
canonical object coordinates
→ object motion/deformation
→ stage pose
→ global setup geometry
→ detector coordinates
→ detector/nuisance model
```

Every parameter belongs to a graph node and has:

```text
value
unit
scale
active/frozen status
prior
bounds / trust radius
gauge policy
observability status
backend support metadata
```

This is how TomoJAX can support many synchrotron beamlines without turning the CLI into a method zoo.

## Default solver: stopped-volume alternating reconstruction

Do not differentiate through the whole reconstruction loop by default. Treat the volume as a stopped latent variable during geometry updates.

Default loop:

```text
for level in [1/4, 1/2, maybe 1]:
    x = reconstruct(projections, geometry, short_schedule)
    geometry = robust_schur_lm_update(x, projections, geometry)
    geometry = canonicalise_gauges(geometry)
    verification = verify_geometry(x, geometry)
    if verification strongly passes:
        skip unnecessary finer alignment

x_final = reconstruct(projections, solved_geometry, final_schedule)
emit artifacts
```

Research mode may later support unrolled or implicit differentiation through reconstruction, but it is not the fast default.

## Geometry optimiser: robust Schur LM/GN

With `x` fixed, residuals for projection `i` depend on global setup `g` and that projection's pose `p_i`, not on every other view's pose. This gives an arrowhead normal-equation structure:

```text
[ H_gg   H_gp1 H_gp2 ... H_gpN ] [ δg  ] = [ b_g  ]
[ H_p1g  H_p1  0    ... 0     ] [ δp1 ]   [ b_p1 ]
[ H_p2g  0     H_p2 ... 0     ] [ δp2 ]   [ b_p2 ]
...
```

Each per-view pose block is only `5×5`. The global setup block is small.

Use Schur complement:

```text
S   = H_gg - Σ_i H_gp_i H_pp_i^-1 H_pg_i
rhs = b_g  - Σ_i H_gp_i H_pp_i^-1 b_p_i

δg   = solve(S, rhs)
δp_i = H_pp_i^-1 (b_p_i - H_pg_i δg)
```

This gives one joint setup+pose update without a large dense solve.

## Continuation replaces grid searches

Large capture range should come from continuation, not from trying discrete centres or angles.

Use three continuation axes:

### Resolution continuation

```text
level 4 or 8:
    coarse volume
    low-frequency residual
    large trust radii
    robust loss
    stronger priors

level 2:
    sharper volume
    weaker priors
    same geometry solver

level 1:
    final reconstruction
    geometry polish only if verification fails
```

### Frequency continuation

Use low-pass residuals first, then introduce band-pass/raw residuals. Geometry should see large structure before high-frequency texture.

### Parameter continuation

All main parameters may be active, but with different trust radii and prior strengths:

```text
early:
    det_u, roll, axis, theta_offset, all-5 pose active
    weak/uncertain DOFs heavily prior-weighted
    det_v frozen or strongly prior-weighted

middle:
    relax pose and setup priors
    activate theta_scale if observable
    activate smooth drift if useful

late:
    sparse jumps, object motion, or deformation only if verification demands
```

## Gauges and canonicalisation

For reconstruction-first TomoJAX, gauges are not moral constraints. They stabilise representation.

After each accepted geometry update:

```text
mean_dx = mean(dx_i)
dx_i -= mean_dx
det_u_px += mean_dx

mean_phi = mean(phi_residual_i)
phi_residual_i -= mean_phi
theta_offset_deg += mean_phi

if det_v_px is active:
    mean_dz = mean(dz_i)
    dz_i -= mean_dz
    det_v_px += mean_dz
```

If a gauge transform changes the realised forward geometry materially, mark it as non-exact and treat it as a soft prior / observability issue.

## Verification-triggered escalation

Do not run full-resolution geometry because it is available. Run it only if needed.

Verification should decide:

```text
accept solved geometry
run level 2 alignment
run one full-resolution polish
fall back to reference/tortoise solver
activate nuisance model
activate object-motion model
declare failure with classification
```

## Backend strategy

Use two backends.

### JAX reference backend

```text
full autodiff
calibrated detector grids
correctness oracle
finite-difference checks
debugging
```

### Pallas fast backend

Targets:

```text
forward projection tiles
backprojection tiles
robust residual reductions
Jᵀr reductions
JᵀJ reductions
mixed setup/pose reductions
```

Do not try to put the optimiser itself into Pallas. Accelerate the expensive math kernels.

Every run records:

```text
requested backend
actual backend
fallback reason
canonical/calibrated detector-grid status
JAX/Pallas agreement status, where applicable
```

## Research modules

Exactly five experimental tracks are worth adding after the core works:

1. `neural_scout`: small neural-field or hash-grid scout at coarse resolution.
2. `learned_autofocus`: auxiliary learned quality/preconditioner signal, never replacing the physics residual.
3. `pnp_final`: plug-and-play / RED final reconstruction with projection-consistency checks.
4. `deformable_object`: object-frame affine/B-spline deformation.
5. `global_uncertainty`: Schur/Laplace/randomised uncertainty around geometry.

## What not to build as the default identity

Avoid default user-facing implementations of:

```text
COR grid search
roll grid search
sinogram symmetry search
entropy/sharpness search
SPSA
CMA-ES
Nelder-Mead
phase correlation default
feature tracking default
```

Developer diagnostics are fine. The public product should remain one gradient-first alignment engine.

## Final default workflow

```text
1. Load and normalise projections.
2. Build geometry graph from metadata and beamline profile.
3. Declare reconstruction-first gauge policy.
4. Level 4 preview reconstruction.
5. Robust Schur LM setup+pose update.
6. Gauge canonicalisation.
7. Reconstruction refresh.
8. Geometry update again.
9. Level 2 continuation if needed.
10. Final full-resolution reconstruction.
11. Verification report and artifacts.
```

The output should contain both:

```text
realised forward geometry
canonical physical decomposition
```

The scientist gets a crisp reconstruction. The developer gets traceability.
